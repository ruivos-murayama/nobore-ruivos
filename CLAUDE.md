# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

**のぼれRUIVOSくん** — a one-touch "sling-and-stick" action game (climb a vertical cave to the exit). Vanilla HTML5 Canvas + JavaScript, **no build step, no external libraries, no npm**. To run it, open `index.html` in a browser; to apply any change, just reload. Scripts load in fixed order via `index.html`: `design.js` → `cave.js` → `game.js`.

It deliberately recreates the *feel* of the mobile game Ordia ("pull to launch", "stick to any surface"), retextured as a stealth-climb (night castle, guard sightlines, cloak). Only the game mechanics are imitated — all code and assets are original.

## Architecture

Three layered globals, wired together at runtime through `window`:

- **`design.js`** → exposes `window.DESIGN` (and `module.exports` for Node). The **single source of truth for all tunable numbers**: physics (in px/s, not px/frame), feel (squash, slow-mo, shake, camera), gimmick params, rules (`launchPerDango`, `lowWarnAt`), stealth, and per-stage `stages[].palette`. `★` comments mark the knobs to reach for first. **When asked to change how the game looks or feels, edit here first** — see `DESIGN.md` for the "I want X → change knob Y" table.
- **`cave.js`** → exposes `window.CAVE` (and Node `module.exports`). Pure geometry: seeded procedural cave generation (`buildCave(gen, COL)`) plus all collision math (`circlePoly`, `pointInPoly`, `segPoly`). Owns terrain **shape and difficulty**. `buildCave` returns `{ walls, hazards, bouncy, boosts, sentries, cloaks, movers, platforms, slipWalls, swings, dango, start, goal, worldH }`. Left/right walls are always continuous so the player can always wall-climb up (anti-softlock).
- **`game.js`** → the whole runtime (IIFE). Reads `DESIGN`/`CAVE`, runs the loop, physics solver, stealth, rendering, audio, the stage-select map, and progress persistence. Aliases at top: `D = DESIGN`. It does not export anything.

**Game loop** (`frame()` near the end of `game.js`): a **fixed-timestep accumulator** at 120 Hz (`physics.fixedStep`) with render interpolation (`render(alpha)`), so behavior is frame-rate independent. `timeScale` drives slow-mo during aim charge; `simTime` advances independently (guard sweeps keep moving even during death animation).

**State machine** — `gameState` ∈ `'title' | 'map' | 'play' | 'clear' | 'allclear'`. The HTML overlays in `index.html` (`#title`, `#clear`, `#allclear`) are shown/hidden per state; `'map'` and `'play'` render to the canvas. `'map'` is the castle-themed stage-select (`drawMap` / `updateMap` / `mapTapAt`, inertial scroll).

**Stages** — `LEVELS[]` in `game.js` is the authoritative list (20 stages across 12 `WORLDS` chapters; 7-2「振り子瀑」 is the **evolution** of the descent stage 7-1「大滑降」 — same high-speed ice-slide, plus the new **swing-blade** gimmick (`swings`, see Gimmicks): centrally-pivoted lethal ice on a pendulum. Wall-side lanes stay outside the swing's reach (`gen.swingReach`<0.5 guarantees an escape lane = anti-softlock), but alternating wall spikes force center crossings, where you must time the swing. 8-1「跳躍祭」 is the explicit North-Star "pure feel" stage, 8-2「乱れ咲き」 its harder sequel, 8-3「百花繚乱」 the spectacular long finale — same catapult/bounce-flow taste, difficulty raised via mastery demands not friction. 9-1「無音」 (chapter 9 月輪) is the **second North Star** — a **basketball free-throw tower** (`gen.shootout` → `buildShootout`): walls removed, a stack of floating pads each with a tight hoop offset up-and-to-the-side (alternating L/R via `hoopOffset`). Hoops use `ang=+π/2` (axis down) so you score only by **dropping in from above** (arc up and over); the rim is **solid from below** (a one-way barrier reflects any upward passage — 「下からは入れない」). Drop in cleanly → **warp up** to the next pad, repeat (zigzag arcs), top hoop slides left-right = goal. `boxSpacing` > max-reach (~210px) so you physically can't fly pad-to-pad — the hoop is the *only* way up (no wall-climb bypass; the flaw in the first 9-1 attempt). Warps update `level.start` = checkpoint; `level.bounds` reflect side-misses back onto the pad. Boosts/bumpers/catapults can coexist densely because boosts are placed in clear lanes that don't enclose targets, plus an engine-level stall-rescue failsafe — see DESIGN.md 🌟北極星 and the 失速レスキュー note). 11-1「凪」 (chapter 11 水鏡) is the **third North Star** (`gen.lake` → `buildLake`) — a moonlit **stone-skip** crossing: walls removed, a stack of alternating L/R shore posts (you stick to their inner vertical face = sideways normal) with a still water surface between each tier; skim the water *shallow and fast* and it skips (とぷん…とぷん) two-or-three times up onto the opposite shore one tier higher, but enter *steep or slow* and you sink (ぽちゃん) for a gentle checkpoint respawn (no death, no tries). A **moonlit lip** (`lips`) hangs at each shore so you can't just lob up and over — the skip is the *only* route (no wall-climb / lob bypass; avoiding the 9-1 v1 flaw where the gimmick was skippable). Each entry has `code` ("1-1"…), `world`, `maxLaunch` (launch-count cap for that stage), and a `gen` object passed straight to `buildCave`. **The `gen` here and the palette order in `design.js stages[]` must stay aligned by index.** Per-stage comments record the BFS-measured minimum launch count; `maxLaunch` is that minimum plus human-error margin.

**Progress** — saved to `localStorage('nobore_progress')` via `loadProgress` / `saveProgress`. Clearing a stage unlocks the next and returns to the map.

## Key mechanics to know before editing

- **Launch budget (縛り)**: each stage caps launches at `LEVELS[].maxLaunch`. Running out before reaching the exit = retry ("もう とべない…", distinct from death). Collecting 💧 (`dango`) refunds `rules.launchPerDango` launches; the count refills fully on every respawn (death never softlocks via the budget).
- **Stick constraint**: from a stuck surface you can only fire within `stick.maxOffNormal` of the surface normal — this forbids straight-up wall-crawling and forces arcing jumps.
- **Goal gate (skill-shot)**: the exit hangs inside a ceiling cup (`cupGate`) with a narrow central slit (`gen.gateHalf`); `physics.goalRadius` is tightened so you must shoot *up through* the opening, not graze it. Later stages harden the gate with moving bumpers, gate-guard sentries, and moving spikes (`gen.bumperMove` / `gateGuard` / `gateMover`).
- **Gimmicks**: bumpers (`bouncy`), updrafts (`boosts`), **catapults** (`catapults`, fixed-vector launch pads — ignore incoming velocity and fire you along a set direction at `gimmick.catapultPower`; the "designed burst" counterpart to skill-based bumpers, chainable via the shared `bumpChain`), **hoops** (`hoops`, basketball rings — two small rim posts with an opening; crossing the opening in the hoop's *forward* (`ang`) direction without grazing a post = スウィッシュ. Shootout hoops use `ang=+π/2` so forward = downward = drop-in-from-above; a *backward* (upward) crossing within the opening is blocked by a one-way barrier (reflect, e≈0.45) = 「下からは入れない」. A clean drop-in **warps** you to the next pad / wins on the goal hoop; grazing a rim post = clank (deflect with `gimmick.hoopRimRestitution`<1, decays). Detection uses the ballistic segment `blob.px,py → x,y`, before teleport gimmicks. A hoop with `warpY` warps; `goal:true`+`move` is the moving goal), moving platforms (`platforms`), slip walls (`slipWalls`, slowly slide down), **swing-blades** (`swings`, lethal ice on a pendulum — pivoted at channel center, the bob swings on an arc `θ = arc·sin(simTime·w + phase)`, position `(x+L·sinθ, y+L·cosθ)`; only the bob is lethal, checked in *every* state like `movers`. Currently descent-only; `gen.swingReach`<0.5 keeps the bob off the walls so a wall lane is always safe = anti-softlock), **water-skip** (water surfaces `skips` plus the moonlit lip `lips` — a reflective non-stick both-sided lob-blocker; `gen.lake` → `buildLake`. Skim a water line shallow-and-fast and you skip (`=skip`, とぷん): the surface gives a damped vertical bounce plus `gimmick.skipLift` up-velocity and keeps your horizontal speed (`skipFric`) so you carry onto the next shore one tier up. Enter steep (angle > `skipMaxAngle`) or slow (speed < `skipMinSpeed`) and you `sink` instead (ぽちゃん) for a gentle checkpoint respawn. Main knobs: `gimmick.skipRest`/`skipFric`/`skipLift`/`skipMaxAngle`/`skipMinSpeed`. The lip both-side-reflects so lobbing over is impossible — the skip is the *only* route, no lob bypass), guards (`sentries`, sweeping vision cones blocked by walls), cloak drops (`cloaks`, temporary invisibility). Counts are set per stage in `gen` (`bouncyCount`, `boostCount`, `catapultCount`, `hoopCount`, `platCount`, `slipCount`, `swingCount`, `sentryCount`, `cloakCount`, `hazardCount`); the descent stage also takes `swingArc`/`swingSpeed`/`swingR`/`swingReach`; the free-throw tower uses `gen.shootout` with `boxCount`/`boxSpacing`/`hoopRise`/`goalMove`/`goalGap` instead; the stone-skip lake uses `gen.lake` with `boxCount`/`tierRise`/`lipGap`/`lipW` instead. Catapults are pure juice — never required for reachability (the BFS solver ignores them), so wall-climb stages stay solvable without them. (Shootout is the exception: there the hoop *is* the only route, guaranteed by `boxSpacing` > max-reach, verified by a bypass sweep rather than the BFS solver.)

## UI styling

`style.css` styles only the HTML chrome (title / buttons / HUD). The `:root` CSS variables (`--bg`, `--wall`, `--accent`, `--blob`, `--danger`) mirror stage palette colors — keep them in rough sync with `design.js` if you change the look.

## Verification

There is no test runner or package.json. Validate changes with:

```
node --check design.js cave.js game.js   # syntax of all three
```

`DESIGN.md` also references Node harnesses (`harness.js` = boots through one frame without exceptions; `solve.js` = BFS-verifies every stage is still beatable under current physics/gen; `minlaunch.js` = measures theoretical minimum launches). **Always run the solver after changing physics** (`gravity`, `launchMul`, `maxPull`, `radius`) or any `gen`/`seed` — new physics can create unsolvable caves; fix by adjusting `gen` (gap / yStep) or the seed. Note: these harness scripts live in `/tmp` (not in the repo, and they hard-code an older project path), so confirm/adjust their `require` paths before relying on them. The stone-skip lake has its own harness `scratchpad/verify-lake-real.js` (real `buildCave` + game-identical physics) which checks the full 11-tier run-through, zero lob bypasses, and zero softlocks.

## Sandbox

`feel.html` / `feel.js` are a standalone feel-tuning playground, independent of the main game — do not assume changes there affect the actual game.

## Docs to read

`DESIGN.md` is the design-system reference: a "what to change for X" table mapped to `design.js` knobs, the easing choices (camera = SmoothDamp critical-damped spring; squash = 2nd-order spring), the recommended order of knobs to tune, and the stage-by-stage difficulty intent. `README.md` covers how to play and the stage list.
